# Plan 09-01: ParamGrid Parser and OptimizerSweep with Rayon Parallelism

## Objective
Implement the parameter grid parser and the core Rayon-parallelized optimizer sweep engine that runs N×M×... strategy combinations concurrently.

## Requirements Addressed
- OPT-01: Parameter grid sweep with TOML config defining param ranges
- OPT-02: Rayon parallel execution — each combo independent, work-stealing across CPU cores
- OPT-03: 720-combo sweep (6×6×5×4 grid) completes in <3 minutes for 4yr data

## Wave Execution

### Wave 1: ParamGrid Data Types
**Files:**
- `apps/quantedge_core/crates/optimizer/src/param_grid.rs`

**Details:**
Define the parameter grid configuration:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ParamRange {
    pub name: String,        // e.g. "sl_value", "strike_offset", "lots"
    pub min: f64,
    pub max: f64,
    pub step: f64,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ParamGrid {
    pub params: Vec<ParamRange>,
}

#[derive(Debug, Clone, Serialize)]
pub struct ParamCombo {
    pub index: usize,
    pub values: HashMap<String, f64>,
}

impl ParamGrid {
    pub fn from_json_str(s: &str) -> Result<Self, String>;
    pub fn total_combos(&self) -> usize;
    pub fn generate_combos(&self) -> Vec<ParamCombo>;
}
```

Combo generation: cartesian product of all param ranges. For a 6×6×5×4 grid → 720 combos.

Supported parameter names (applied to strategy TOML before each run):
- `sl_value` → legs[0].stop_loss_value
- `target_value` → overall.overall_target_value
- `strike_offset` → legs[0].strike_offset (as i32)
- `lots` → legs[0].lots (as u32)
- `entry_time_offset` → shifts entry_time by N minutes
- `exit_time_offset` → shifts exit_time by N minutes

### Wave 2: OptimizerSweep with Rayon
**Files:**
- `apps/quantedge_core/crates/optimizer/src/sweep.rs`

**Details:**
```rust
pub struct OptimizerResult {
    pub combo_index: usize,
    pub params: HashMap<String, f64>,
    pub metrics: MetricsResult,
    pub trade_count: u32,
}

pub struct OptimizerSweep;

impl OptimizerSweep {
    /// Run all parameter combinations in parallel using Rayon.
    pub fn run(
        base_config: &StrategyConfig,
        bars: &[SimBar],
        grid: &ParamGrid,
        lot_size: u32,
    ) -> Vec<OptimizerResult>;
}
```

Implementation:
1. `grid.generate_combos()` → Vec<ParamCombo>
2. Use `rayon::par_iter()` to run each combo:
   a. Clone `base_config`
   b. Apply param overrides from combo.values to the cloned config
   c. `SimRunner::run(&modified_config, bars, lot_size)`
   d. `MetricsEngine::compute(...)` on the result
   e. Return `OptimizerResult`
3. Collect, sort by Sharpe descending
4. Return top results

Config override function:
```rust
fn apply_overrides(config: &mut StrategyConfig, overrides: &HashMap<String, f64>) {
    for (name, value) in overrides {
        match name.as_str() {
            "sl_value" => {
                for leg in &mut config.legs {
                    leg.stop_loss_value = *value;
                }
            }
            "target_value" => {
                config.overall.overall_target_value = Some(*value);
            }
            "strike_offset" => {
                for leg in &mut config.legs {
                    leg.strike_offset = *value as i32;
                }
            }
            "lots" => {
                for leg in &mut config.legs {
                    leg.lots = *value as u32;
                }
            }
            _ => {} // Unknown param — skip
        }
    }
}
```

### Wave 2: DuckDB Storage for Optimizer Results
**Files:**
- `apps/quantedge/lib/quantedge/duck/writer.ex` — add/update `insert_optimizer_results/2`
- `apps/quantedge/lib/quantedge/duck/reader.ex` — add `get_optimizer_results/1`

**Details:**
DuckDB table:
```sql
CREATE TABLE IF NOT EXISTS optimizer_results (
  run_id VARCHAR,
  combo_index INTEGER,
  params JSON,
  sharpe DOUBLE,
  total_pnl DOUBLE,
  max_dd_pct DOUBLE,
  trade_count INTEGER,
  win_rate DOUBLE,
  profit_factor DOUBLE,
  cagr DOUBLE
);
```

Writer: `insert_optimizer_results(run_id, results_list)` — batch insert using prepared statement.
Reader: `get_optimizer_results(run_id)` — fetch all, sorted by sharpe DESC. Return `[%{combo_index, params, sharpe, ...}]`.

### Wave 3: OptimizerWorker Update with PubSub Progress
**Files:**
- `apps/quantedge/lib/quantedge/workers/optimizer_worker.ex` — full update

**Details:**
Update worker to:
1. Load strategy and param_grid from the OptimizerRun Postgres record
2. Call `NIF.run_optimizer(strategy.config_toml, Jason.encode!(run.param_grid))`
3. Parse result JSON
4. Store results via `Duck.Writer.insert_optimizer_results(run_id, results)`
5. Update Postgres: status=completed, completed_combos=count
6. Broadcast to `optimizer:updates` channel:
   - `{:optimizer_progress, run_id, percent}` (approximated)
   - `{:optimizer_completed, run_id, top_results}`

Progress approximation: since Rayon runs all combos in parallel, broadcast estimated progress based on elapsed time vs expected time (combo_count × avg_single_run_time).

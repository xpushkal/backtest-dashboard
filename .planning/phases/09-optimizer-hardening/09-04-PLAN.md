# Plan 09-04: Edge Case Test Suite (9 Edge Cases from PRD)

## Objective
Build a comprehensive Rust test suite covering all 9 edge cases specified in OPT-05, ensuring the simulation engine handles degenerate inputs gracefully without panics.

## Requirements Addressed
- OPT-05: Edge case hardening: 0 trades, first/last bar entry, transition week, lot size change, zero volume, IV=0, max concurrent, re-entry exhaustion

## Wave Execution

### Wave 1: Zero-Trade and Boundary Cases
**Files:**
- `apps/quantedge_core/crates/core/tests/edge_cases.rs`

**Details:**
Test cases 1-3:

**1. Zero trades scenario:**
- Entry time is after the last bar time → 0 trades
- Verify: RunResult.trades is empty, snapshots exist, no panic
- Metrics compute returns zeroed values

**2. First-bar entry:**
- Entry time = first bar time (09:15 or 09:20)
- Verify: trade opens on first bar, no index underflow
- Check entry_price is from the first available bar

**3. Last-bar exit:**
- Exit time = last bar time (15:30)
- Verify: EndOfData exit reason triggered
- Position held to absolute end without panic

### Wave 2: Market Transition and Lot Size Edge Cases
**Files:**
- `apps/quantedge_core/crates/core/tests/edge_cases.rs` — extend

**Details:**
Test cases 4-6:

**4. Transition week (BankNifty Oct-Nov 2024 lot size change):**
- Generate bars where lot_size changes from 15 to 30 mid-series
- Strategy uses lot_size=15 (passed at run time)
- Verify: all trades use the same lot_size consistently
- No mid-trade lot size confusion

**5. Lot size change mid-trade:**
- Strategy enters position with lot_size=15
- Bars continue but lot_size parameter doesn't change mid-run
- Verify: exit uses same lot_size as entry (no mismatch)
- PnL calculation: (exit - entry) × direction × lots × lot_size

**6. Zero volume bar:**
- Generate bars where some have close=0.0
- Verify: engine skips zero-price entries (no divide-by-zero)
- No NaN in PnL or metrics

### Wave 3: IV, Concurrency, and Re-Entry Edge Cases
**Files:**
- `apps/quantedge_core/crates/core/tests/edge_cases.rs` — extend

**Details:**
Test cases 7-9:

**7. IV = 0:**
- Generate bars with iv=0.0 (spot is always present)
- Verify: if IV surface not used, backtest runs normally
- If Greeks computation: handle IV=0 gracefully (BS model returns 0 for greeks)

**8. Max concurrent trades limit:**
- Set `max_concurrent_trades = 1`
- Generate bars where multiple entries would trigger on same day
- Verify: only 1 trade opens, subsequent entries skipped
- Position count never exceeds limit

**9. Re-entry exhaustion:**
- Set `max_reentries = 2` with re-entry mode = ASAP
- Trigger SL 3 times in rapid succession
- Verify: exactly 2 re-entries (attempts 1, 2), third attempt blocked
- `reentry_attempt` field: 0, 1, 2 (no 3)
- Trade count = 3 (initial + 2 re-entries)

### Wave 4: Integration Edge Cases
**Files:**
- `apps/quantedge_core/crates/core/tests/edge_cases.rs` — extend

**Details:**
Additional robustness tests:

**10. Empty bar data:**
- Pass empty Vec<SimBar> to SimRunner::run
- Verify: returns RunResult with empty trades, 0 total_bars

**11. Single-bar data:**
- Only 1 bar at entry time
- Verify: entry but no exit (EndOfData or no trade)

**12. Weekend gap in dates:**
- Friday → Monday with no Saturday/Sunday bars
- Verify: strategy handles gap correctly, no missing date panic

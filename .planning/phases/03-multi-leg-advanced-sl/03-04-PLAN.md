## Objective

Implement OCO (One-Cancels-Other) between SL and target at the leg level.
When SL fires on a leg, the target for that leg is cancelled. When target
fires, the SL is cancelled. Enforce the full 5-level exit priority chain.

## must_haves

- OCO: if SL triggers on a leg, target check is skipped for same bar
- OCO: if target triggers on a leg, SL check result is discarded
- Exit priority strictly enforced: per-leg SL → combined SL → per-leg target → overall target → time
- OCO is implicit (not a separate state) — priority chain naturally implements it
- Integration tests proving OCO behavior with conflicting signals

<action>
OCO is inherently handled by the priority chain — the existing `check_exits`
already returns the FIRST match. When per-leg SL fires (priority 1), the method
returns immediately, so per-leg target (priority 3) is never checked.

Verify and add code comment documenting OCO behavior:
```rust
/// Check exit conditions in strict priority order.
///
/// OCO (One-Cancels-Other) is implicit in the priority chain:
/// - If per-leg SL fires (priority 1), we return immediately — target (priority 3) is never checked
/// - If combined SL fires (priority 2), per-leg target (priority 3) is cancelled
/// - This means SL always takes precedence over target on the same bar
fn check_exits(...) -> Option<ExitReason> { ... }
```

Add explicit `ExitReason` tracking: when a per-leg SL fires, the exit reason should
indicate WHICH leg triggered it. Add to `ExitReason`:
```rust
pub enum ExitReason {
    StopLoss,
    Target,
    TimeExit,
    EndOfData,
    CombinedSl,     // NEW: overall/combined SL
    CombinedTarget, // NEW: overall/combined target
}
```
</action>
<acceptance_criteria>
- `ExitReason` has `CombinedSl` and `CombinedTarget` variants
- `check_exits` returns `CombinedSl` when overall SL fires (not `StopLoss`)
- `check_exits` returns `CombinedTarget` when overall target fires (not `Target`)
- Per-leg SL still returns `StopLoss`
- Per-leg target still returns `Target`
- `cargo check -p quantedge-core` compiles
</acceptance_criteria>
</task>

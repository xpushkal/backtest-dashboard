# Plan 01-05: Data Validation Script and Integration Tests

<objective>
Create the data validation script (scripts/validate_data.py) that performs 6 integrity checks on the converted Parquet files. The script exits 0 on clean data and exits 1 with detailed error report on failures. This is the quality gate before any phase can use the data.
</objective>

<type>execute</type>
<files>

<action>
Create `scripts/validate_data.py` with 6 validation checks.

The script must:
1. Accept `--data-dir` argument (default: `data/parquet/`)
2. Accept `--symbol` argument (optional, validates all if omitted)
3. Load Parquet files using Python polars
4. Run all 6 checks, collecting pass/fail results
5. Print a summary table at the end
6. Exit 0 if all pass, exit 1 if any fail

**Check 1: Weekly Cutoff**
- For each symbol, verify no Parquet files exist in the `weekly/` partition after the transition date
- BANKNIFTY: no weekly data after 2024-10-31
- NIFTY: no weekly data after 2024-10-02
- SENSEX: no weekly data after 2024-09-17
- Implementation: scan each file's max date column; flag if any exceeds cutoff

**Check 2: Date Gaps**
- For each symbol, extract all unique trading dates, sort them
- Check for gaps >3 calendar days (accounts for weekends + holidays)
- Flag gaps >5 calendar days as warnings, >10 as errors
- Exclude known long holidays (Diwali, etc.) from error flagging — use a configurable holiday list or just warn

**Check 3: Duplicates**
- Check for duplicate rows with same (timestamp, strike, option_type)
- If duplicates found, report count and first 5 examples
- This is a hard error — duplicates corrupt backtesting results

**Check 4: IV Coverage**
- Count total bars and bars where `iv` is null, NaN, 0, or negative
- Compute coverage percentage: `valid_iv_bars / total_bars * 100`
- Pass if ≥95%, fail if <95%
- Report per-symbol coverage

**Check 5: Spot Continuity**
- For each symbol, compute daily spot price (using last bar of each day)
- Check day-over-day change: `abs(spot_today / spot_yesterday - 1)`
- Flag if change >10% (highly unusual for index options)
- This catches data corruption or missing corporate action adjustments

**Check 6: Lot Size Coverage**
- Load config/lot_sizes.toml
- For each (symbol, trading_date) in the data, verify a lot size entry exists
- Flag any dates that fall outside configured lot size ranges

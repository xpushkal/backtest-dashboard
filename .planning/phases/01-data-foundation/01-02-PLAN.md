<type>execute</type>
<files>

</files>
<action>
Create `scripts/csv_to_parquet.py` — the CSV to Parquet conversion script.

The script must:
1. Accept CLI args: `--input-dir` (path to raw CSVs), `--output-dir` (path to Parquet output, default: `data/parquet/`)
2. For each CSV in input-dir, infer the symbol (banknifty/nifty/sensex) from filename
3. Parse the `date` column as Date type
4. For each row, determine if it belongs to `weekly/` or `monthly/` partition using the expiry transition dates from `config/expiry_calendar.toml`
5. Group by (symbol, expiry_type, year, month)
6. Write each group as a separate Parquet file: `{output_dir}/{symbol}/{expiry_type}/{year}/{month:02}.parquet`
7. Use Snappy compression (Polars default)
8. Print summary: files created, total rows, per-symbol counts

Expected column schema (validate before writing):
- timestamp: Utf8
- date: Date
- time: Utf8
- weekday: Utf8
- option_type: Utf8 (CE/PE)
- strike_label: Utf8
- strike_offset: Int32
- moneyness: Utf8
- open: Float64
- high: Float64
- low: Float64
- close: Float64
- volume: Int64
- strike: Float64
- oi: Float64
- spot: Float64
- iv: Float64

The transition logic:
- BANKNIFTY: dates <= 2024-10-31 go to weekly/, dates >= 2024-11-01 go to monthly/
- NIFTY: dates <= 2024-10-02 go to weekly/, dates >= 2024-10-03 go to monthly/
- SENSEX: dates <= 2024-09-17 go to weekly/, dates >= 2024-09-18 go to monthly/
- Monthly data exists for the FULL period (all years). The monthly/ partition contains ALL rows for dates that fall in a "monthly" expiry regime.
</action>

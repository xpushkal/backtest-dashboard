# Plan 08-04: NIF Wrapper (run_portfolio), Oban Worker, Integration Tests

## Objective
Wire the Rust portfolio engine to Elixir via NIF, implement the PortfolioWorker for Oban background execution, store results in DuckDB, and validate with end-to-end integration tests.

## Requirements Addressed
- NIF-03: Rustler NIF for run_portfolio (multi-strategy simultaneous)
- STR-03: PortfolioWorker with PubSub status updates

fn run_portfolio(strategies_json: String, opts_json: String) -> Result<String, String> {
    // 1. Parse PortfolioRequest from JSON

### Wave 2: Elixir PortfolioWorker Update
**Files:**
- `apps/quantedge/lib/quantedge/workers/portfolio_worker.ex` — full implementation
- `apps/quantedge/lib/quantedge/runs.ex` — add portfolio run management

**Details:**
Update `PortfolioWorker` to:
1. Accept `%{portfolio_run_id, strategies_json, opts_json}`
2. Call `NIF.run_portfolio(strategies_json, opts_json)`
3. Parse the JSON result
4. Store per-strategy trades and equity in DuckDB via `Duck.Writer`
5. Store portfolio metrics in DuckDB
6. Update portfolio run status in Postgres
7. Broadcast progress via PubSub: `portfolio:{run_id}`

Add to Runs context:
- `create_portfolio_run(attrs)` — creates Postgres record
- `complete_portfolio_run(run_id, result_summary)` — stores result
- `enqueue_portfolio(run_id)` — enqueues Oban job

### Wave 3: DuckDB Storage for Portfolio Results
**Files:**
- `apps/quantedge/lib/quantedge/duck/writer.ex` — add portfolio tables
- `apps/quantedge/lib/quantedge/duck/reader.ex` — add portfolio queries

**Details:**
New DuckDB tables:
```sql
CREATE TABLE IF NOT EXISTS portfolio_equity (
  run_id VARCHAR,
  date DATE,
  portfolio_equity DOUBLE,
  strategy_name VARCHAR,
  strategy_equity DOUBLE
);

CREATE TABLE IF NOT EXISTS portfolio_metrics (
  run_id VARCHAR,
  metric_name VARCHAR,
  metric_value DOUBLE
);
```

Writer functions:
- `insert_portfolio_equity(run_id, equity_data)`
- `insert_portfolio_metrics(run_id, metrics_map)`

Reader functions:
- `get_portfolio_equity(run_id)` — combined + per-strategy curves
- `get_portfolio_metrics(run_id)` — all portfolio-level metrics
- `get_portfolio_correlation(run_id)` — correlation matrix JSON

### Wave 4: Integration Tests
**Files:**
- `apps/quantedge/test/quantedge/portfolio_test.exs`

**Details:**
Tests (run with `mix test --only integration`):
1. **NIF Round-trip**: Create 2-strategy JSON → call NIF → verify response JSON has expected keys
2. **Performance**: 3-strategy portfolio 4yr < 5 seconds
3. **Margin Skip**: Configure low capital → verify margin_skips in result > 0
4. **Allocation**: 60/40 split → each strategy's allocated_capital matches
5. **Worker Flow**: Enqueue portfolio → verify Postgres status transitions (pending → running → completed)
6. **DuckDB Storage**: After worker completes, verify equity data retrievable from DuckDB
7. **Correlation Matrix**: Verify NxN matrix is symmetric, diagonal = 1.0

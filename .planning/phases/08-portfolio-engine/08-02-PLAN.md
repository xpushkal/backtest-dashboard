# Plan 08-02: Simplified SPAN MarginModel with Margin Check at Entry

## Objective
Implement a simplified SPAN margin model that computes required margin per trade and rejects entries when portfolio margin would exceed available capital.

## Requirements Addressed
- PORT-02: Simplified SPAN margin correctly computed
- PORT-03: Trades skipped and logged as "margin_skip" when margin insufficient

## Wave Execution

### Wave 1: MarginModel Implementation
**Files:**
- `apps/quantedge_core/crates/portfolio/src/margin.rs`

**Details:**
Simplified SPAN margin formula per leg:

```
margin_required = max(
    3 × premium × lots × lot_size,           // 3x premium rule
    index_factor × spot × lots × lot_size × 0.12  // 12% of notional
)
```

Where:
- `premium` = option entry price
- `lots` = number of lots
- `lot_size` = underlying lot size (15 for BN, 75 for NF, etc.)
- `spot` = current spot price
- `index_factor` = 1.0 (BankNifty, Nifty) or 1.0 (Sensex) — configurable

Struct:

```rust
pub struct MarginModel {
    index_factor: f64,  // default 1.0
}

pub struct MarginResult {
    pub required: f64,
    pub rule_used: MarginRule,  // ThreeXPremium or NotionalPercent
}

pub enum MarginRule {
    ThreeXPremium,
    NotionalPercent,
}

impl MarginModel {
    pub fn new(index_factor: f64) -> Self;

    /// Compute margin for a single leg entry.
    pub fn compute_leg_margin(
        &self,
        premium: f64,
        spot: f64,
        lots: u32,
        lot_size: u32,
    ) -> MarginResult;

    /// Compute total margin for a multi-leg strategy entry.
    /// For hedged positions (buy leg + sell leg on same strike), apply 70% hedge benefit.
    pub fn compute_strategy_margin(
        &self,
        legs: &[(f64, f64, u32, u32, PositionSide)],  // (premium, spot, lots, lot_size, side)
    ) -> f64;
}
```

Hedge benefit:
- If strategy has both buy and sell legs on same option_type → reduce total margin by 30% (hedge benefit factor 0.7)
- Otherwise, sum individual leg margins

### Wave 2: PortfolioMarginTracker
**Files:**
- `apps/quantedge_core/crates/portfolio/src/margin.rs` — extend

**Details:**
Track margin usage across the portfolio in real-time:

```rust
pub struct PortfolioMarginTracker {
    total_capital: f64,
    margin_model: MarginModel,
    active_margins: HashMap<String, f64>,  // strategy_name -> current margin
    peak_margin: f64,
    margin_skips: Vec<MarginSkip>,
}

pub struct MarginSkip {
    pub date: NaiveDate,
    pub time: NaiveTime,
    pub strategy_name: String,
    pub required_margin: f64,
    pub available_margin: f64,
    pub reason: String,
}

impl PortfolioMarginTracker {
    pub fn new(total_capital: f64, model: MarginModel) -> Self;

    /// Check if a new entry can be accepted. Returns Ok(margin) or Err(MarginSkip).
    pub fn check_entry(
        &mut self,
        strategy_name: &str,
        required_margin: f64,
        date: NaiveDate,
        time: NaiveTime,
    ) -> Result<f64, MarginSkip>;

    /// Release margin when a position is closed.
    pub fn release_margin(&mut self, strategy_name: &str, amount: f64);

    /// Current margin utilization (0.0 - 1.0).
    pub fn utilization(&self) -> f64;

    /// Total peak margin used.
    pub fn peak_margin(&self) -> f64;

    /// All margin skips that occurred.
    pub fn skips(&self) -> &[MarginSkip];
}
```

### Wave 3: Integration with PortfolioEngine + Tests
**Files:**
- `apps/quantedge_core/crates/portfolio/src/engine.rs` — add margin tracking
- `apps/quantedge_core/crates/portfolio/tests/margin_tests.rs`

**Details:**
Modify `PortfolioEngine::run()` to:
1. Create `PortfolioMarginTracker` with `total_capital`
2. Before each strategy opens a position: `check_entry()`
3. If margin insufficient: log `MarginSkip`, skip the trade (set exit_reason = MarginSkip)
4. When position closes: `release_margin()`
5. Add to `PortfolioResult`:
   - `peak_margin_used: f64`
   - `capital_efficiency: f64` = total_pnl / peak_margin_used
   - `margin_skips: Vec<MarginSkip>`
   - `avg_margin_utilization: f64`

Tests:
- 2 strategies, capital = 100K, one strategy requiring 80K margin → second strategy entry skipped
- Hedge benefit reduces margin correctly (buy + sell same type → 70% of sum)
- `margin_skip` count matches expected skips
- `peak_margin` never exceeds `total_capital`

## Objective

Parse TOML strategy files into strongly-typed Rust structs. This is the entry point
for the simulation engine — every backtest starts by loading a `StrategyConfig`.
Phase 2 only needs ATM offset strike mode and basic SL/target; other modes are
stubbed for Phase 3.

## must_haves

- StrategyConfig struct with all fields from strategy TOML spec
- LegConfig struct with option_type, position, lots, strike_mode, SL, target
- OverallConfig struct with overall SL/target settings
- Enum types: OptionType, PositionSide, StrikeMode, SlType, ExitReason, SlippageModel
- TOML parsing with clear validation errors for missing/invalid fields
- Example strategy TOML file that validates correctly

```rust
impl StrategyConfig {
    pub fn from_toml(path: &str) -> Result<Self, Box<dyn std::error::Error>> {
        let content = std::fs::read_to_string(path)?;
        Self::from_toml_str(&content)
    }

    pub fn from_toml_str(content: &str) -> Result<Self, Box<dyn std::error::Error>> {
        let config: StrategyConfig = toml::from_str(content)?;
        config.validate()?;
        Ok(config)
    }

    fn validate(&self) -> Result<(), Box<dyn std::error::Error>> {
        if self.legs.is_empty() {
            return Err("Strategy must have at least one leg".into());
        }
        if self.capital <= 0.0 {
            return Err("Capital must be positive".into());
        }
        if self.entry_time >= self.exit_time {
            return Err("entry_time must be before exit_time".into());
        }
        for (i, leg) in self.legs.iter().enumerate() {
            if leg.lots == 0 {
                return Err(format!("Leg {} must have lots > 0", i).into());
            }
        }
        Ok(())
    }
}
```

Use intermediate serde structs if needed for TOML field name mapping (e.g.,
`stop_loss_type` string → `SlType` enum). The `entry_time` and `exit_time` fields
parsed from "HH:MM" strings using chrono `NaiveTime::parse_from_str(s, "%H:%M")`.
</action>

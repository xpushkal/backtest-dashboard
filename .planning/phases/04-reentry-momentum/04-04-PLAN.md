# Plan 04-04: Momentum-Confirm Re-entry + End-to-End Integration

## Goal
Wire the momentum engine into the re-entry state machine so that
`MomentumConfirm` mode only re-enters when the momentum filter aligns with
the position direction. Run full end-to-end integration tests validating
all 4 re-entry modes with momentum interaction.

## Context
- Plan 04-01 created the state machine with `WaitingForMomentum` state
- Plan 04-02 wired ASAP/SameTime/AfterNBars into the runner
- Plan 04-03 created the momentum engine with per-bar signals
- This plan connects momentum signals to the WaitingForMomentum â†’ Ready transition

## Tasks

### 1. Add momentum signal feeding to ReEntryTracker

```rust
impl ReEntryTracker {
    /// Feed a momentum signal. Only relevant in WaitingForMomentum state.
    /// For sell positions: Bearish signal confirms re-entry
    /// For buy positions: Bullish signal confirms re-entry
    pub fn feed_momentum(&mut self, signal: MomentumSignal, position_side: PositionSide) {
        if let ReEntryState::WaitingForMomentum { attempt } = &self.state {
            let confirmed = match position_side {
                PositionSide::Sell => signal == MomentumSignal::Bearish,
                PositionSide::Buy => signal == MomentumSignal::Bullish,
            };
            if confirmed {
                self.state = ReEntryState::Ready { attempt: *attempt };
            }
        }
    }
}
```

### 2. Wire momentum engine into SimRunner

In the main loop, before re-entry check:
1. Pre-compute momentum signals at start of `run()` if any leg has momentum configured
2. On each bar, look up the signal for the current (date, time)
3. Feed signal to trackers in `WaitingForMomentum` state

```rust
// At start of run()
let momentum = MomentumEngine::from_config_and_bars(config, bars);

// In main loop, after tick()
if let Some(ref engine) = momentum {
    let signal = engine.signal_at(group_index);
    for (i, tracker) in trackers.iter_mut().enumerate() {
        tracker.feed_momentum(signal, config.legs[i].position);
    }
}
```

### 3. Entry gating by momentum filter

If `momentum_filter` is configured and `momentum_filter_enabled` is true,
block initial entry when momentum signal contradicts position direction:
- Sell leg + Bullish signal â†’ skip entry
- Buy leg + Bearish signal â†’ skip entry

This is separate from re-entry confirmation and applies to initial entries too.

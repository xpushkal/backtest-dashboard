# Plan 04-01: Re-entry State Machine

## Goal
Implement the full re-entry state machine as a standalone module with 5 states
(Idle → Cooling → WaitingForMomentum → Ready → Exhausted) and configurable
max_attempts, cooldown, and trigger conditions (SL hit, target hit).

## Context
The existing `LegConfig` already has stubbed re-entry fields:
- `reentry_on_sl: bool` — trigger re-entry when SL fires
- `reentry_on_target: bool` — trigger re-entry when target fires
- `reentry_mode: String` — needs to be converted to typed enum
- `reentry_cooldown_bars: u32` — bars to wait before re-entry
- `reentry_max_attempts: u32` — max re-entries before Exhausted

## Tasks

### 1. Add `ReEntryMode` and `ReEntryState` enums to config.rs

```rust
/// Re-entry mode after SL/target hit.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum ReEntryMode {
    Asap,              // re-enter at next bar open
    SameTime,          // re-enter at same clock time next trading day
    AfterNBars,        // wait N bars then re-enter
    MomentumConfirm,   // wait for momentum filter to confirm
}

impl Default for ReEntryMode {
    fn default() -> Self { ReEntryMode::AfterNBars }
}
```

Replace `reentry_mode: String` in `LegConfig` with `reentry_mode: ReEntryMode`.
Remove `default_reentry_mode()` helper.

### 2. Create `reentry.rs` module

```rust
/// Re-entry state machine per leg.
#[derive(Debug, Clone)]
pub enum ReEntryState {
    /// No re-entry pending.
    Idle,
    /// Cooling down — waiting for cooldown_bars to elapse.
    Cooling { bars_remaining: u32, attempt: u32 },
    /// Waiting for momentum filter confirmation (MomentumConfirm mode only).
    WaitingForMomentum { attempt: u32 },
    /// Cleared to re-enter on next entry opportunity.
    Ready { attempt: u32 },
    /// All re-entry attempts exhausted.
    Exhausted,
}
```

Key methods:
- `ReEntryTracker::new(config: &LegConfig) -> Self`
- `ReEntryTracker::on_exit(exit_reason: ExitReason) -> ()` — triggers state transition from Idle → Cooling/Ready
- `ReEntryTracker::tick() -> ()` — called every bar, decrements cooldown counter
- `ReEntryTracker::should_reenter() -> bool` — returns true when state is `Ready`
- `ReEntryTracker::confirm_reentry() -> ()` — transitions Ready → Idle (reset for next exit)
- `ReEntryTracker::is_exhausted() -> bool`

State transitions:
```
Idle → (SL/Target hit, reentry_on_sl/target enabled)
  ├── Asap mode          → Ready { attempt: 1 }
  ├── AfterNBars mode    → Cooling { bars_remaining: N, attempt: 1 }
  ├── SameTime mode      → Ready { attempt: 1 } (runner gates on time match)
  └── MomentumConfirm    → WaitingForMomentum { attempt: 1 }

Cooling → (tick each bar, bars_remaining -= 1)
  └── bars_remaining == 0 → Ready { attempt }

WaitingForMomentum → (momentum_signal received)
  └── Ready { attempt }

Ready → (re-entry executed)
  └── attempt < max_attempts → Idle (awaiting next exit)
  └── attempt >= max_attempts → Exhausted

Exhausted → terminal state
```

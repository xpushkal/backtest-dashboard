# Plan 04-02: Re-entry Mode Integration into SimRunner

## Goal
Wire the `ReEntryTracker` into the main simulation loop so that after a position
closes (SL or target), the runner checks re-entry conditions and opens a new
position when the tracker says `should_reenter() == true`.

## Context
Currently `SimRunner::run()` sets `position = None` after closing, and only opens
a new position when `time == entry_time`. Re-entry must bypass the normal entry time
gate and open at the bar dictated by the re-entry mode.

## Tasks

### 1. Add per-leg `ReEntryTracker` to runner state

Store a `Vec<ReEntryTracker>` alongside the position. When a position closes:
1. For each leg, call `tracker.on_exit(exit_reason)` based on leg config
2. If the exit was EndOfData or TimeExit with no re-entry configured, don't arm

### 2. Integrate re-entry into the main loop

After exit/close block, add a re-entry check:

```rust
// After position = None (close)
// Check if any leg tracker is Ready
let should_reenter = trackers.iter().any(|t| t.should_reenter());

if should_reenter && position.is_none() {
    match primary_mode {
        ReEntryMode::Asap => {
            // Re-enter immediately at current bar
            if let Some(pos) = Self::try_open(config, bars, indices, lot_size, date, time) {
                position = Some(pos);
                for t in &mut trackers { t.confirm_reentry(); }
            }
        }
        ReEntryMode::SameTime => {
            // Only re-enter if current time matches original entry_time
            if time == config.entry_time() {
                // ... open position
            }
        }
        ReEntryMode::AfterNBars => {
            // Tracker handles cooldown; when Ready, open at any bar
            // ... open position
        }
        ReEntryMode::MomentumConfirm => {
            // Handled in Plan 04-04
        }
    }
}
```

### 3. Tick trackers every bar

When position is None and trackers are in Cooling state, call `tick()` on each bar.

### 4. Track re-entry count in ClosedTrade

Add `reentry_attempt: u32` to `ClosedTrade` to record which attempt produced this trade.
Default to 0 for initial entry.

### 5. Integration tests

| Test | Description |
|------|-------------|
| `test_asap_reentry_opens_immediately` | SL hit → new position opens on same bar |
| `test_after_n_bars_reentry` | SL hit → position opens after N bars |
| `test_same_time_reentry_next_day` | SL hit → position opens next day at entry_time |
| `test_max_attempts_stops_reentry` | After max_attempts re-entries, no more positions open |
| `test_no_reentry_on_time_exit` | TimeExit with reentry_on_sl=true → no re-entry (wrong trigger) |
| `test_reentry_on_target` | Target hit with reentry_on_target=true → re-entry fires |

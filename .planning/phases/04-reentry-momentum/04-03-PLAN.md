# Plan 04-03: Momentum Filters

## Goal
Implement 4 momentum filters as a standalone module. Each filter takes a spot
price series and emits a signal (Bullish/Bearish/Neutral). Filters are
pre-computed where possible for O(1) per-bar lookups.

## Context
Momentum filters serve two purposes:
1. **Entry filter** — block initial entry if momentum is against direction
2. **Re-entry confirmation** — for `MomentumConfirm` mode, only re-enter when
   momentum aligns with position direction

The runner needs to access momentum signals at each (date, time) key.

### 2. Create `momentum.rs` module

```rust
/// Momentum signal output.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum MomentumSignal {
    Bullish,
    Bearish,
    Neutral,
}

/// Pre-computed momentum signals indexed by bar position.
pub struct MomentumEngine {
    signals: Vec<MomentumSignal>,  // one per unique (date, time) group
}
```

#### RSI Implementation
Standard RSI(period) on spot close prices:
```
gain_avg = EMA of gains over period
loss_avg = EMA of losses over period
RS = gain_avg / loss_avg
RSI = 100 - (100 / (1 + RS))
Signal: RSI > bullish_above → Bullish, RSI < bearish_below → Bearish, else Neutral
```

#### EMA Cross Implementation
Compute fast and slow EMA on spot prices:
```
Signal: fast_ema > slow_ema → Bullish, fast_ema < slow_ema → Bearish
```

#### Range Breakout Implementation (RE-04: O(N) setup, O(1) lookup)
Pre-compute opening range (high/low of spot from market open to range_end_time)
for each trading day in a single pass:
```
Pre-compute:
  for each day:
    range_high = max(spot) for bars where time <= range_end_time
    range_low  = min(spot) for bars where time <= range_end_time
    store in HashMap<NaiveDate, (f64, f64)>

Per bar (O(1) lookup):
  if time > range_end_time:
    if spot > range_high → Bullish
    if spot < range_low  → Bearish
    else → Neutral
  else → Neutral (still in range formation)
```

#### Supertrend Implementation
Standard supertrend using ATR-based bands:
```
ATR = SMA of true_range over period
upper_band = (high + low) / 2 + multiplier * ATR
lower_band = (high + low) / 2 - multiplier * ATR
(simplified for spot data: use spot ± ATR bands)
Signal: spot > supertrend_line → Bullish, else → Bearish
```
Note: SimBar currently only has `close` and `spot`. For supertrend, we'll add
`high` and `low` fields to SimBar (optional, defaulting to close).

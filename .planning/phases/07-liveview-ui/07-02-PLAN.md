# Plan 07-02: Dashboard Landing Page

## Objective
Build the main dashboard LiveView showing system overview: recent runs, strategy count, quick-launch actions, and system health indicators.

## Requirements Addressed
- UI-14: Dashboard landing page with recent runs and quick-launch

## Wave Execution

### Wave 1: Dashboard LiveView
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/dashboard_live.ex` — REPLACE stub
- `apps/quantedge_web/lib/quantedge_web/live/dashboard_live.html.heex` — Template

**Details:**
Dashboard sections (top-to-bottom):
1. **Hero Stats Row** (4 stat cards):
   - Total Strategies (count from Postgres)
   - Total Runs (count from Postgres)
   - Best Sharpe (max from completed runs)
   - Total PnL (sum from completed runs)

2. **Quick Actions Bar:**
   - "New Strategy" button → navigates to `/strategies/new`
   - "Quick Backtest" button → modal with strategy picker + date range
   - "Import TOML" button → file upload modal

3. **Recent Runs Table** (last 10):
   - Columns: Strategy Name, Underlying, Date Range, Status (badge), PnL (colored), Sharpe, Duration, Actions
   - Click row → navigate to `/runs/:id`
   - Status column: live-updating via PubSub subscription

4. **System Status Panel** (bottom):
   - Data coverage per underlying (BANKNIFTY: 2021-01-01 to 2024-12-31)
   - DuckDB storage size
   - Rust NIF health check (calls `load_data_summary` to verify NIF is operational)

### Wave 2: PubSub Integration
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/dashboard_live.ex` — Add handle_info callbacks

**Details:**
- On mount, subscribe to `"runs:updates"` topic
- `handle_info({:run_completed, run_id, summary}, socket)` → update recent runs list in place
- `handle_info({:run_started, run_id}, socket)` → add running indicator
- Debounce list refresh to avoid excessive re-renders

## Verification
- [ ] Dashboard loads with stat cards showing real counts
- [ ] Recent runs table populated from Postgres
- [ ] Quick action buttons navigate correctly
- [ ] PubSub subscription receives run updates
- [ ] Empty state shown when no runs exist

## UAT Criteria
- Dashboard renders in under 200ms
- Real-time updates appear without page refresh
- Stat cards display formatted numbers (₹ symbol, comma separation)

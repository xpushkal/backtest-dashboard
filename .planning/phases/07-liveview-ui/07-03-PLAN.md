# Plan 07-03: Strategy Builder LiveView

## Objective
Interactive multi-leg strategy builder with TOML generation, import/export, and full CRUD.

## Requirements Addressed
- UI-01: Strategy builder LiveView (create/edit multi-leg strategies)
- UI-02: Strategy list with CRUD operations and TOML import/export
- DSL-02, DSL-03: TOML strategy DSL support

## Wave Execution

### Wave 1: Strategy List (Index LiveView)
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/strategy_live/index.ex`
- `apps/quantedge_web/lib/quantedge_web/live/strategy_live/index.html.heex`

**Details:**
- List all strategies in a card grid (not table — each strategy gets a visual card)
- Card shows: name, underlying badge (BN/NF/SX color-coded), leg count, last modified
- Filter by underlying (tabs: All | BankNifty | Nifty | Sensex)
- "New Strategy" button opens modal form
- Each card has: Edit, Duplicate, Export TOML, Delete actions
- Delete requires confirmation modal
- Import TOML button: file upload → parse → create strategy

### Wave 2: Strategy Builder Form (Modal/Page)
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/strategy_live/form_component.ex`
- `apps/quantedge_web/lib/quantedge_web/live/strategy_live/form_component.html.heex`

**Details:**
Strategy form fields:
1. **Header:** Strategy name (text input), Underlying (select: BANKNIFTY/NIFTY/SENSEX)
2. **Overall Config:** Capital, Entry time, Exit time, Lot size
3. **Legs Section** (dynamic — add/remove legs):
   Each leg has:
   - Option type (CE/PE select)
   - Position (buy/sell toggle)
   - Lots (number input)
   - Expiry (weekly/monthly select)
   - Strike mode (atm_offset/fixed select)
   - Strike offset (number input, shown when atm_offset)
   - SL Type (select from 7 types)
   - SL Value (number input)
   - Target Type (select)
   - Target Value (number input)
   - Trailing SL toggle + trail config
   - Re-entry toggle + re-entry config (mode, max_attempts)

4. **"Add Leg" button** — appends new leg form with defaults
5. **"Remove Leg" button** — removes leg (min 1 leg required)
6. **Preview panel:** Live TOML preview as user edits (right sidebar)
7. **Save button** — generates TOML from form state, saves to Postgres

### Wave 3: TOML Import/Export
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/strategy_live/index.ex` — add import handlers

**Details:**
- **Export:** Download `.toml` file with strategy config (Content-Disposition header)
- **Import:** Upload `.toml` file → parse with NimbleTOML → validate → create strategy
- **Round-trip test:** Import → edit → export produces valid TOML that re-imports identically
- Handle parse errors gracefully with flash messages

## Verification
- [ ] Strategy CRUD works (create, read, update, delete)
- [ ] Multi-leg form adds/removes legs dynamically
- [ ] TOML preview updates in real-time as form changes
- [ ] Import/export round-trip produces identical TOML
- [ ] Validation errors shown inline (name required, at least 1 leg)
- [ ] Delete confirmation modal works

## UAT Criteria
- User can create a 4-leg iron condor strategy via the GUI
- TOML export of created strategy is valid and parseable by Rust kernel
- Import of example_iron_condor.toml populates form correctly

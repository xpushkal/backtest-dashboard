# Plan 07-01: Phoenix Layout, Navigation & Design System

## Objective
Establish the QuantEdge design system, app shell layout (sidebar + content area), and navigation infrastructure. Every subsequent LiveView will compose inside this shell.

## Requirements Addressed
- Foundation for all UI-* requirements

## Wave Execution

### Wave 1: Design System CSS (app.css)
**Files:**
- `apps/quantedge_web/assets/css/app.css` — Full design system

**Details:**
Create a dark-mode-first design system optimized for data-dense financial dashboards:
- **Color palette:** Dark backgrounds (`#0a0a0f`, `#12121a`, `#1a1a2e`), accent cyan (`#00d4ff`), accent green (`#00e676`), accent red (`#ff3d3d`), muted text (`#8b8fa3`), bright text (`#e8eaf0`)
- **Typography:** `Inter` from Google Fonts (400/500/600/700). Monospace: `JetBrains Mono` for numbers/metrics
- **Spacing:** 4px grid system (0.25rem increments)
- **Cards:** `.card` — `background: #12121a`, `border: 1px solid #1e1e2e`, `border-radius: 12px`, subtle box-shadow
- **Buttons:** `.btn-primary` (cyan gradient), `.btn-secondary` (ghost), `.btn-danger` (red)
- **Inputs:** Dark background inputs with focus ring (cyan glow)
- **Tables:** `.data-table` with alternating row backgrounds, sticky headers
- **Badges:** `.badge-success`, `.badge-warning`, `.badge-error`, `.badge-info`
- **Utility classes:** `.text-mono` (numbers), `.text-profit` (green), `.text-loss` (red), `.flex-between`, `.grid-2`, `.grid-3`, `.grid-4`
- **Animations:** `@keyframes fadeIn`, `@keyframes slideUp`, `@keyframes pulse-glow`

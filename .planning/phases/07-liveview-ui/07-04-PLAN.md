# Plan 07-04: Run Configuration & Execution with Real-Time Progress

## Objective
Run configuration form, Oban job enqueueing, and real-time progress via PubSub.

## Requirements Addressed
- UI-03: Run configuration (date range, capital, brokerage, slippage)
- UI-10: Real-time progress bar during backtest execution

## Wave Execution

### Wave 1: Run List LiveView
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/run_live/index.ex`
- `apps/quantedge_web/lib/quantedge_web/live/run_live/index.html.heex`

**Details:**
- Table of all backtest runs, sortable by date, status, PnL
- Status column with live-updating badges (pending → running → completed/failed)
- Columns: Strategy, Underlying, Date Range, Capital, Status, PnL, Sharpe, Actions
- Filter by status (All / Running / Completed / Failed)
- "New Run" button opens configuration modal
- Click row navigates to results view (`/runs/:id`)

### Wave 2: Run Configuration Modal
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/run_live/config_component.ex`
- `apps/quantedge_web/lib/quantedge_web/live/run_live/config_component.html.heex`

**Details:**
Configuration form fields:
1. Strategy picker (select from saved strategies)
2. Date range (date_from / date_to pickers)
3. Capital (number input, default 100000)
4. Brokerage per order (number, default 20)
5. Slippage points (number, default 0.5)
6. "Run Backtest" button — creates BacktestRun in Postgres, enqueues Oban job

### Wave 3: Real-Time Progress
**Files:**
- `apps/quantedge_web/lib/quantedge_web/live/run_live/index.ex` — PubSub handlers

**Details:**
- Subscribe to `run:{run_id}` on mount for each visible run
- Progress bar component shows percentage (broadcast from BacktestWorker)
- Status transitions animate: pending(gray) → running(cyan pulse) → completed(green) / failed(red)
- On completion, PnL and Sharpe columns update in-place without page reload
- Toast notification on completion: "Backtest completed — PnL: +₹45,000"

## Verification
- [ ] Run list shows all runs from Postgres
- [ ] New run modal creates record and enqueues Oban job
- [ ] Progress bar updates in real-time via PubSub
- [ ] Status badges transition correctly
- [ ] Clicking completed run navigates to results

## UAT Criteria
- User can configure and launch a backtest in under 30 seconds
- Progress is visible without page refresh
- Multiple simultaneous runs show independent progress
